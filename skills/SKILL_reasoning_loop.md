## Description: Reasoning loop for multi-step task planning.

## Prompt:
Using filesystem tools, read all .md files from vault_path='AI_Employee_Vault/Needs_Action'. For each file:

### Step 1: Analyze the Task
Read the file content and frontmatter. Identify:
- Task type (email, whatsapp, file_drop, etc.)
- Priority level
- Required actions

### Step 2: Reason Step-by-Step
For each task, think through:
1. What is the goal?
2. What actions are needed?
3. What is the order of execution?
4. Are there dependencies?
5. What could block progress?

### Step 3: Create Plan File
Write to `AI_Employee_Vault/Plans/{task_id}_Plan.md` with structure:

```markdown
---
task_id: {filename_without_extension}
type: {task_type}
priority: {priority}
status: planning
created: {timestamp}
max_iterations: 3
current_iteration: 1
---

## Task Summary
{Brief description of what needs to be done}

## Reasoning
{Step-by-step analysis of the task}

## Action Plan

- [ ] Step 1: {First action}
- [ ] Step 2: {Second action}
- [ ] Step 3: {Third action}
- [ ] Step 4: {Fourth action}
- [ ] Step 5: {Fifth action}

## Dependencies
{List any prerequisites or blockers}

## Notes
{Additional context or considerations}

---
*Generated by SKILL_reasoning_loop.md*
```

### Step 4: Iterate (Simulated Loop)
After creating initial plan:
1. Check if plan is complete (all steps defined)
2. If incomplete or unclear, create revised plan as `{task_id}_Plan_v2.md`
3. Maximum 3 iterations per task
4. Mark final plan with `status: ready`

### Example Output

For an email about an invoice:
```markdown
---
task_id: EMAIL_19c6859be1a38185
type: email
priority: high
status: ready
created: 2026-02-17T02:30:00
max_iterations: 3
current_iteration: 1
---

## Task Summary
Process urgent invoice email from vendor regarding payment.

## Reasoning
1. Email is marked urgent and relates to payment
2. Need to verify invoice details before processing
3. May require approval from finance team
4. Should respond to sender acknowledging receipt

## Action Plan

- [ ] Step 1: Read full email content and extract invoice details
- [ ] Step 2: Verify invoice against purchase orders
- [ ] Step 3: Forward to finance team for approval
- [ ] Step 4: Send acknowledgment reply to sender
- [ ] Step 5: Update tracking spreadsheet

## Dependencies
- Access to purchase order database
- Finance team approval

## Notes
- Invoice due date: Check email for specifics
- Mark as high priority in finance queue

---
*Generated by SKILL_reasoning_loop.md*
```

## Output Rules
- One plan file per Needs_Action item
- Use clear, actionable step descriptions
- Include estimated effort if determinable
- Flag items needing human review
