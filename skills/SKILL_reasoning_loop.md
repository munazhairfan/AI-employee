## Description: Reasoning loop for multi-step task planning with anti-hallucination safeguards.

## ANTI-HALLUCINATION RULES (CRITICAL)

1. **Extract ONLY** - Never invent information not present in source
2. **Cite Sources** - Every fact must include exact source text
3. **Confidence Score** - Rate confidence (0-100%) for each extraction
4. **Mark Unknown** - If info not found, mark as "UNKNOWN - needs clarification"
5. **No Assumptions** - Do not assume amounts, dates, names, or details

## Prompt:
Using filesystem tools, read all .md files from vault_path='AI_Employee_Vault/Needs_Action'. For each file:

### Step 1: Analyze the Task
Read the file content and frontmatter. Identify:
- Task type (email, whatsapp, file_drop, etc.)
- Priority level
- Required actions
- **Source content** (exact text from file)

### Step 2: Reason Step-by-Step with Source Citation
For each task, think through:

1. **What is the goal?**
   - Cite: "Exact text from source that indicates this goal"
   - Confidence: 0-100%

2. **What actions are needed?**
   - Cite: "Exact text from source that implies this action"
   - Confidence: 0-100%

3. **What is the order of execution?**
   - Cite: "Exact text or logical reasoning"
   - Confidence: 0-100%

4. **Are there dependencies?**
   - Cite: "Exact text mentioning dependencies" OR "None mentioned in source"
   - Confidence: 0-100%

5. **What could block progress?**
   - Cite: "Exact text mentioning blockers" OR "None mentioned in source"
   - Confidence: 0-100%

### Step 4: Iterate (Simulated Loop)
After creating initial plan:
1. Check if plan is complete (all steps defined)
2. Check if all fields have source citations
3. Check if confidence scores are valid (0-100%)
4. If incomplete or unclear, create revised plan as `{task_id}_Plan_v2.md`
5. Maximum 3 iterations per task
6. Mark final plan with `status: ready`

## Confidence Score Guidelines

| Confidence | Meaning | Example |
|------------|---------|---------|
| 100% | Explicitly stated | "$500" → amount=500 |
| 90-99% | Strongly implied | "send bill" → create_invoice (90%) |
| 70-89% | Reasonable inference | "urgent" → priority=high (85%) |
| 50-69% | Weak inference | "soon" → specific date (60%) |
| 0-49% | Unknown/Assumed | Customer name not mentioned (0%) |

**RULE:** Fields with <80% confidence MUST trigger human review
