## Description: Poll Pending_Approval and execute only if approved (draft-only handler).

## Prompt:
Use filesystem on vault_path='AI_Employee_Vault'. List all .md in /Pending_Approval. For each, read content. If line '- [ ] Approve' is marked with [x] (case-sensitive, x in []), extract action_type (from file name or frontmatter), content (email body, post text, invoice data). Prepare for manual execution. If '- [ ] Reject' is marked with [x], move to /Rejected/{file}.md and append 'Rejected: [extract reason below Reject]'. If no mark, do nothing. No direct execution without approval. Output ONLY full .md content. No additions, no hallucinations.

---

## Execution Rules

### Step 1: Scan Pending_Approval Folder
```
- List all *.md files in AI_Employee_Vault/Pending_Approval/
- For each file, read full content
- Parse frontmatter to extract metadata
- Check Approval Actions checkboxes
```

### Step 2: Check Approval Status
```
Search for exact markers (case-sensitive):
- '- [x] Approve' → Prepare for manual execution
- '- [x] Reject' → Move to Rejected with reason
- '- [ ] Approve' (unchecked) → Skip (do nothing)
- No checkbox → Skip (do nothing)
```

### Step 3: Prepare Approved Actions (Manual Execution)

**Email Drafts:**
```
Extract from draft:
- To: {email address}
- Subject: {subject line}
- Body: {email content}

Action: Copy to clipboard, open Gmail/Outlook, paste, send manually
Output: "Email draft ready. Details copied to clipboard. Please send manually."
```

**Invoice Drafts:**
```
Extract from draft:
- Client: {client name}
- Amount: {amount}
- Description: {description}

Action: Open Odoo ERP, navigate to Invoices → Create, fill details manually
Output: "Invoice draft ready. Please create manually in Odoo."
```

**LinkedIn/Facebook Drafts:**
```
Extract from draft:
- Post Text: {post content}

Action: Copy to clipboard, open LinkedIn/Facebook, paste, post manually
Output: "Social post draft ready. Text copied to clipboard. Please post manually."
```

**WhatsApp Reply Drafts:**
```
Extract from draft:
- Reply: {reply content}

Action: Copy to clipboard, open WhatsApp, paste, send manually
Output: "WhatsApp reply ready. Text copied to clipboard. Please send manually."
```

### Step 4: Handle Rejections
```
If '- [x] Reject' found:
1. Extract rejection reason (text below or near Reject marker)
2. Move file to AI_Employee_Vault/Rejected/{filename}.md
3. Append to file content:
   'Rejected: [extracted reason]'
4. Log rejection to Logs/approval_log.json
```

### Step 5: Post-Execution Actions

**For Approved + Prepared:**
```
1. Move file to AI_Employee_Vault/Done/{filename}.md
2. Update status in file to 'executed_manually'
3. Add execution timestamp
4. Log to Logs/approval_log.json
```

**For Rejected:**
```
1. Move to AI_Employee_Vault/Rejected/
2. Append rejection reason
3. Log to Logs/approval_log.json
```

**For Skipped (No Mark):**
```
- Leave file in Pending_Approval/
- No action taken
- Will be re-checked on next poll
```

---

## Approval File Template

```markdown
---
type: {action_type}_draft
status: pending_review
generated_at: {timestamp}
---

## Content

{Email body, post text, invoice details, or reply text}

---

## Approval Actions

- [ ] Approve (execute manually)
- [ ] Reject (add your reason below)
- [ ] Edit (add your changes/notes below)

---
*Draft generated by SKILL_*.md - No API calls made*
```

**Logs/approval_log.json:**
```json
[
  {
    "timestamp": "2026-02-22T10:30:00",
    "file": "Email_Draft_2026-02-22_10-30-00.md",
    "action_type": "email",
    "decision": "approved | rejected | pending",
    "executed_manually": true,
    "rejection_reason": null
  }
]
```

---

## Safety Constraints

1. **NEVER** execute without '- [x] Approve' marker
2. **NEVER** modify approval file before execution
3. **ALWAYS** log all actions to approval_log.json
4. **ALWAYS** move file after processing (Done/Rejected)
5. **NEVER** call MCP/API endpoints (draft-only workflow)
6. **ALWAYS** extract exact values from file content
7. **IF** manual execution required, copy to clipboard and guide user
8. **NEVER** hallucinate endpoints or parameters

---

## Polling Interval

- Check Pending_Approval every 60 seconds
- Process all files with marks in each poll
- Skip files without marks (leave for next poll)
