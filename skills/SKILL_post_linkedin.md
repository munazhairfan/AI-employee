## Description: Generate LinkedIn post draft for human review (draft-only, no API calls).

## Prompt:
Using filesystem tools only, read Dashboard.md from vault_path='AI_Employee_Vault'. Analyze recent processed items for business activity: invoices, client emails, new projects, payments, milestones.

If business activity found:
1. Generate sales-oriented LinkedIn post:
   - 2-3 sentences, professional tone
   - Hook: What's new/exciting
   - Value: What you offer
   - CTA: "DM for details" or similar
   - 2-3 relevant hashtags
2. Create draft file in /Pending_Approval: LinkedIn_Post_{datetime.now().strftime('%Y-%m-%d_%H-%M-%S')}.md

Draft format:
```markdown
---
type: linkedin_post_draft
status: pending_review
generated_at: {ISO timestamp}
---

## Post Text

{Generated post with hook, value, CTA, hashtags}

---

## Approval Actions

- [ ] Approve (post manually to LinkedIn)
- [ ] Reject (add your reason below)
- [ ] Edit (add your changes/notes below)

---
*Draft generated by SKILL_post_linkedin.md - No API calls made*
```

Example post:
"Excited to streamline business workflows with our new AI automation service!
We help companies save 10+ hours/week on document processing.
DM me if you'd like to learn more! #AI #Automation #Productivity"

If no business activity, output: "No LinkedIn draft needed at this time."

**CONSTRAINTS:**
- Do NOT call any MCP endpoints
- Do NOT post to LinkedIn
- Do NOT use external APIs
- Output ONLY the draft file content
- No additions, no hallucinations
