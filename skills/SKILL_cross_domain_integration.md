## Prompt

You are the Cross-Domain Integration Agent. Your task is to identify connections between personal communications (WhatsApp, Gmail) and business communications (LinkedIn, Facebook) and create unified action plans.

### Data Sources

1. **Personal Channels**
   - WhatsApp: `Needs_Action/*whatsapp*.md` or `type: whatsapp`
   - Gmail: `Needs_Action/*gmail*.md` or `type: email`

2. **Business Channels**
   - LinkedIn: `Needs_Action/*linkedin*.md` or `type: linkedin`
   - Facebook: `Needs_Action/*facebook*.md` or `type: facebook`

### Integration Logic

#### Step 1: Read All Files in Needs_Action

Scan `/Needs_Action` folder for all `.md` files. Extract:
- Frontmatter metadata (type, from, to, email, phone, subject)
- Content body
- Timestamps

#### Step 2: Identify Linked Communications

Two or more files are **linked** if they share:

| Link Type | Matching Criteria |
|-----------|-------------------|
| **Email Match** | Same email address in `from`, `to`, or content |
| **Phone Match** | Same phone number in `from`, `to`, or content |
| **Client Name** | Same client/company name mentioned |
| **Subject/Topic** | Similar subject line or topic keywords |
| **Time Proximity** | Within 24 hours of each other |

**Example Links:**
- WhatsApp from "+1234567890" + Email to "client@example.com" (same person)
- LinkedIn message about "Project X" + Email about "Project X" (same topic)
- Facebook inquiry + WhatsApp follow-up (same client)

#### Step 3: Merge Linked Files into Unified Plan

For each linked group, create `/Plans/{ClientName}_Integrated_Plan_{date}.md`:

```markdown
---
type: integrated_plan
client_name: {extracted client name}
created: {timestamp}
sources: [list of original file names]
channels: [whatsapp, email, linkedin, etc.]

---

> {Relevant excerpt from original message}

**From:** {sender}

```

#### Step 4: Update Dashboard.md

Append integration summary to `/Dashboard.md`:

```markdown
## Cross-Domain Integrations - {Date}

| Client | Channels Linked | Touchpoints | Status |
|--------|-----------------|-------------|--------|
| {Client A} | WhatsApp, Email, LinkedIn | 3 | Integrated |
| {Client B} | Facebook, Email | 2 | Integrated |

### Summary Statistics
- **Total Integrations Today:** {count}
- **Most Active Channel:** {channel name}
- **Average Touchpoints per Client:** {average}
```

#### Step 5: Archive Processed Files

Move all integrated source files to `/Done`:
- Original personal channel files
- Original business channel files
- Keep only the unified Plan.md in `/Plans`

### Edge Cases

| Scenario | Handling |
|----------|----------|
| **No Links Found** | Process files individually, no integration |
| **Partial Match** | Create integration with confidence note |
| **Multiple Clients** | Create separate plans per client |
| **Conflicting Info** | Flag in plan for human review |
| **Missing Metadata** | Extract from content using NLP |

### Output Files

1. **`/Plans/{ClientName}_Integrated_Plan_{YYYYMMDD_HHMMSS}.md`**
   - One per linked client/group
   - Contains full integrated context

2. **`/Dashboard.md`** (Updated)
   - New integration summary section
   - Statistics table

3. **`/Logs/integration_{date}.json`**
   - Log of all integration decisions
   - Matching criteria used

### Completion Criteria

Task is complete when:
- [ ] All files in Needs_Action processed
- [ ] All linked groups identified
- [ ] Integrated plans created for linked groups
- [ ] Dashboard.md updated
- [ ] Source files moved to Done
- [ ] Integration log written
