## Description: Generate email draft for human review with anti-hallucination safeguards (draft-only, no API calls).

## ANTI-HALLUCINATION RULES (CRITICAL)

1. **Extract ONLY** - Never invent email addresses, names, amounts, or dates
2. **Cite Sources** - Every extracted field must include exact source text
3. **Confidence Score** - Rate confidence (0-100%) for each extraction
4. **Mark Unknown** - If info not found, mark as "UNKNOWN - needs clarification"
5. **No Assumptions** - Do not assume recipient, subject, or content

## Prompt:
Using filesystem tools only, read files from vault_path='AI_Employee_Vault/Needs_Action'. Look for email-related content or requests to send emails. Generate email drafts for human review and approval.

### Step 1: Identify Email Requests

Scan all .md files in /Needs_Action for:
- `type: email` in frontmatter
- Email indicators: 'send email', 'send an email', 'email to', 'reply to', 'compose email'
- Email patterns: email addresses, subject lines, To:/From: fields
- **Record exact source text for each indicator found**

### Step 2: Extract Email Details with Source Citations

For each email request, extract with citations:

| Field | Extraction Rule | Source Citation Required |
|-------|-----------------|-------------------------|
| **To:** | Email address or name | Quote exact text mentioning recipient |
| **CC:** | CC recipients | Quote exact text mentioning CC |
| **Subject:** | Subject line | Quote exact text or infer from context |
| **Body:** | Email content | Quote key points from source |
| **Priority:** | urgent/high/normal | Quote urgency indicators |

**Confidence Scoring:**
- 100%: Explicitly stated (e.g., "email john@example.com")
- 90-99%: Strongly implied (e.g., "send to John" with known email)
- 70-89%: Reasonable inference (e.g., "reply to vendor")
- 0-49%: Unknown/not mentioned

**RULE:** If any critical field (To, Subject) has <80% confidence, mark draft for mandatory human review.

### Step 3: Generate Email Draft with Confidence Tracking
