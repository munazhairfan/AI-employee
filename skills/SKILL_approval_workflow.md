## Description: Human-in-the-loop approval workflow - draft generation only (no MCP calls).

## Prompt:
Using filesystem tools, monitor and process actions in vault_path='AI_Employee_Vault' that require human approval before execution. Generate draft approval requests without executing any actions.

### Trigger Conditions
Create approval draft in Pending_Approval if ANY of these apply:
- **Financial**: Amount > $500 (check for $, USD, payment keywords)
- **Sensitive**: Contains confidential, legal, HR keywords
- **External Communication**: Sending emails/posts to clients/partners
- **High Priority**: Marked as urgent + involves external parties

### Step 1: Identify Actions Needing Approval

Read all Plan.md files from `AI_Employee_Vault/Plans/`. For each plan:

1. Check action type:
   - `send_email` → Requires approval draft
   - `linkedin_post` → Requires approval draft
   - `payment` → Requires approval draft if > $500
   - `invoice` → Requires approval draft

2. Check content for sensitivity:
   - Financial amounts ($500+, invoice, payment, transfer)
   - Legal terms (contract, agreement, terms, liability)
   - Confidential (NDA, private, confidential, internal only)
   - HR matters (salary, review, termination, hiring)

### Step 2: Create Approval Draft File

For actions needing approval, create `AI_Employee_Vault/Pending_Approval/{task_id}_Approval.md`:

```markdown
---
type: approval_request
original_file: {original_plan_path}
status: pending_approval
created: {timestamp}
action_type: {email|post|payment|invoice|other}
---

# Approval Request: {Title}

## Action Details

**Type:** {action_type}
**Priority:** {priority}
**To/Recipient:** {recipient}
**Subject:** {subject}
**Amount:** {amount if financial}

## Content

{Full email body, post text, invoice details, or payment info}

## Why Approval Required
- {Reason 1}
- {Reason 2}

---

## Approval Actions

- [ ] Approve (execute action manually after approval)
- [ ] Reject (add your reason below)
- [ ] Edit (add your changes/notes below)

---
*Draft generated by SKILL_approval_workflow.md - No MCP calls made*
```

### Step 3: Log Draft Creation

Append to `AI_Employee_Vault/Approval_Log.md`:

```markdown
| Timestamp | Action | Type | Status | Notes |
|-----------|--------|------|--------|-------|
| {timestamp} | {task_id} | {action_type} | Draft Created | Pending human approval |
```

## Output Rules
- One approval draft file per sensitive action
- Always include approve/reject/edit checkboxes
- Log all draft creations to Approval_Log.md
- **NEVER** call MCP endpoints
- **NEVER** execute actions directly
- **NEVER** modify files without approval mark

## Constraints
- Draft-only: No API calls, no MCP calls, no execution
- Human must manually execute approved actions
- Output ONLY the draft file content
- No additions, no hallucinations
