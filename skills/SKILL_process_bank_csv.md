## Description: Process bank CSV and generate Odoo invoice drafts for human review (draft-only, no API calls).

## Prompt:
Using filesystem tools only, scan vault_path='AI_Employee_Vault/Drop' and vault_path='AI_Employee_Vault/Accounting' for CSV files. Parse bank statement CSV files and generate invoice drafts for payment transactions.

### Step 1: Find and Parse CSV

Locate latest CSV file in Drop/ or Accounting/ folders. Parse with these expected columns:
- `date` - Transaction date
- `description` - Transaction description
- `amount` - Transaction amount
- `type` - Transaction type (credit/debit)

### Step 2: Identify Payment Transactions

For each transaction:
- Check if amount > 0 (payment received)
- Search description for payment keywords: 'payment', 'transfer', 'deposit', 'credit', 'invoice', 'inv', 'received'
- Extract client name from description (capitalize words, known clients)

### Step 3: Generate Invoice Drafts

For each matched payment, create draft in /Pending_Approval: Invoice_Draft_{client_name}_{datetime.now().strftime('%Y-%m-%d_%H-%M-%S')}.md

Draft format:
```markdown
---
type: invoice_draft
status: pending_review
generated_at: {ISO timestamp}
source_file: {CSV file name}
transaction_date: {transaction date}
---

## Invoice Details

**Client Name:** {extracted client name}
**Amount:** PKR {amount:,.2f}
**Description:** {transaction description}
**Source:** Bank CSV - {CSV file name}

## Suggested Odoo Action

Create invoice in Odoo ERP with:
- Partner: {client name}
- Amount: PKR {amount}
- Description: Payment received - {description}

---

## Approval Actions

- [ ] Approve (create invoice manually in Odoo)
- [ ] Reject (add your reason below)
- [ ] Edit (add your changes/notes below)

---
*Draft generated by SKILL_process_bank_csv.md - No MCP calls made*
```

### Step 4: Update Dashboard

Append summary to /Dashboard.md:
```markdown
## Bank CSV Processing - {timestamp}

| Client | Amount | Status |
|--------|--------|--------|
| {client} | PKR {amount} | Draft Created |
```

### Step 5: Log Unmatched Transactions

For transactions without clear client/payment match, log to Logs/csv_unmatched.log:
```json
{
  "timestamp": "{ISO timestamp}",
  "source_file": "{CSV name}",
  "transaction": {transaction details},
  "reason": "No client match or payment keyword"
}
```

**CONSTRAINTS:**
- Do NOT call Odoo MCP endpoints
- Do NOT create invoices automatically
- Do NOT modify Odoo database
- Output ONLY the draft file content
- No additions, no hallucinations
